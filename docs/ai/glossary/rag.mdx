# RAG

**RAG(Retrieval-Augmented Generation, 검색 증강 생성)** 는 질문과 관련된 자료를 먼저 검색한 뒤, 검색 결과를 LLM의 문맥에 넣어 답을 생성하는 방식입니다.

```text
사용자 질문 → 관련 자료 검색 → 프롬프트에 자료 추가 → LLM이 답변 생성
```

기본 LLM은 학습 이후에 생긴 정보나 사내 문서를 저절로 알지 못합니다. RAG를 사용하면 문서를 모델에 다시 학습시키지 않고도 최신 자료나 조직 내부 자료를 근거로 답하게 할 수 있습니다. 답변에 사용한 출처를 함께 제시하기도 쉽습니다.

RAG가 사실성을 자동으로 보장하지는 않습니다. 검색 결과가 부정확하거나 질문과 관련이 없으면 답변도 잘못될 수 있고, 올바른 자료를 제공해도 LLM이 내용을 잘못 해석할 수 있습니다. 따라서 검색 품질과 최종 답변을 모두 검증해야 합니다.

RAG는 모델의 지식이나 행동 자체를 바꾸는 **파인 튜닝(fine-tuning)** 과 달리, 답을 생성하는 시점에 참고 문맥을 제공하는 방법입니다.
