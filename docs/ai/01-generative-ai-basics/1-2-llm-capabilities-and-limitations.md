# 1-2. LLM의 강점과 한계

LLM은 언어 패턴을 바탕으로 다양한 작업을 수행할 수 있지만 모든 답의 정확성을 보장하지는 않습니다. 잘하는 일과 부족한 일을 구분하면 LLM을 어디에 활용하고 무엇을 검증해야 하는지 판단하기 쉬워집니다.

## LLM이 잘하는 것

LLM은 언어 패턴을 바탕으로 내용을 **생성하고 변환하는 일**에 강합니다.

### 글을 만들고 다듬기

- 게시글, 이메일, 보고서의 초안 작성
- 맞춤법 교정과 문체 변경
- 긴 글의 요약과 핵심 내용 정리
- 번역과 쉬운 표현으로 바꾸기

### 형식을 바꾸기

- 자연어 요구사항을 표나 JSON 같은 구조로 변환
- 여러 문서에서 공통 항목 추출
- 정해진 양식에 맞춰 내용을 재구성

### 아이디어와 예시 만들기

- 주제에 대한 아이디어 확장
- 개념을 설명하는 비유와 예시 생성
- 여러 대안의 장단점 초안 작성

### 코드 작업 보조하기

- 짧은 코드와 테스트 코드 초안 생성
- 익숙하지 않은 코드 설명
- 오류가 발생할 만한 지점과 수정 방향 제안

LLM은 특히 사용자가 충분한 문맥과 원하는 출력 형식을 제공했을 때 더 안정적으로 동작합니다. 참고 자료, 예시, 제약 조건을 함께 주면 막연한 한 문장만 주었을 때보다 원하는 결과에 가까워집니다.

## LLM이 못하거나 주의해야 하는 것

LLM의 답은 자연스럽게 보인다는 것과 사실이라는 것이 별개입니다.

### 사실을 항상 보장하지 못한다

LLM은 그럴듯한 다음 토큰을 선택하기 때문에, 모르는 내용도 자신 있는 문장으로 만들어낼 수 있습니다. 존재하지 않는 인물, 논문, 함수나 출처를 제시하는 현상을 흔히 **환각(hallucination)** 이라고 부릅니다.

따라서 중요한 사실, 숫자, 인용문, 법률이나 의료 정보는 원문과 신뢰할 수 있는 출처로 다시 확인해야 합니다.

### 최신 정보와 비공개 정보를 저절로 알지 못한다

기본 모델이 아는 정보는 학습 데이터와 학습 시점에 제한됩니다. 오늘의 뉴스, 사내 문서, 개인 데이터는 검색 도구나 별도의 데이터 연결 없이 알 수 없습니다.

### 정확한 계산과 엄격한 규칙 수행에 약할 수 있다

산술 계산, 글자 수 세기, 복잡한 조건을 빠짐없이 적용하는 작업은 작은 실수에도 결과가 틀립니다. 계산기, 코드 실행기, 데이터베이스 같은 도구로 검증하는 편이 안전합니다.

### 긴 작업에서 일관성을 잃을 수 있다

입력이 모델이 한 번에 처리할 수 있는 **컨텍스트 윈도우(context window)** 를 넘으면 앞부분을 참고하지 못합니다. 범위 안에 있더라도 긴 문서의 모든 조건을 같은 비중으로 기억하거나 여러 단계의 계획을 끝까지 정확히 수행한다는 보장은 없습니다.

### 질문이 모호하면 의도와 다른 답을 만들 수 있다

LLM은 사용자의 숨은 목적을 완전히 알 수 없습니다. 대상 독자, 목적, 분량, 포함하거나 제외할 내용을 알려주지 않으면 일반적이지만 쓸모가 적은 답이 나올 수 있습니다.

## LLM을 잘 활용하는 방법

LLM은 정답을 보관한 백과사전보다 **빠르게 초안을 만들고, 주어진 정보를 정리하며, 도구 사용을 연결하는 협업자**에 가깝습니다.

- 목적과 독자를 구체적으로 알려줍니다.
- 필요한 배경 자료와 좋은 결과의 예시를 제공합니다.
- 큰 작업은 중간 결과를 확인할 수 있는 작은 단계로 나눕니다.
- 중요한 사실과 계산은 외부 출처나 도구로 검증합니다.
- 개인정보와 회사의 기밀 정보는 사용 중인 서비스의 데이터 정책을 확인한 뒤 입력합니다.

결국 LLM을 잘 쓰는 핵심은 모든 답을 그대로 믿는 것이 아니라, **생성과 판단은 LLM의 도움을 받고 검증은 적절한 도구와 사람이 맡는 것**입니다.

## 정리

LLM은 글쓰기, 요약, 변환과 아이디어 생성처럼 언어를 다루는 작업에 강합니다. 반면 사실성, 최신성, 정확한 계산과 긴 작업의 일관성을 스스로 보장하지는 못합니다.

따라서 LLM은 결과를 그대로 사용하는 정답 기계보다, 사람이 방향을 제시하고 결과를 검증하면서 함께 사용하는 보조 도구에 가깝습니다.
