# 1-1. LLM이란 무엇인가?

LLM은 **Large Language Model**, 즉 **대규모 언어 모델**의 약자입니다.

많은 양의 텍스트를 학습해 문장 안의 패턴을 익히고, 주어진 문맥 다음에 올 가능성이 높은 **토큰(token)** 을 하나씩 예측하는 인공지능 모델입니다. 토큰은 모델이 글을 처리하는 기본 단위로, 한 단어나 글자와 정확히 일치하지 않고 단어의 일부나 문장 부호가 될 수도 있습니다.

예를 들어 다음 문장이 주어졌다고 생각해 봅시다.

LLM은 학습 과정에서 익힌 언어 패턴과 현재 문맥을 바탕으로 `서울`이 다음에 올 가능성이 높다고 계산합니다. 그리고 새로 생성한 토큰을 다시 입력에 포함해 그다음 토큰을 예측합니다. 이 과정을 반복하면서 문장과 문단을 만듭니다.

겉으로는 질문의 답을 데이터베이스에서 찾아오는 것처럼 보이지만, 기본적인 LLM은 저장된 문장을 그대로 검색하는 시스템이 아닙니다. 학습한 패턴을 **모델의 파라미터(parameter)** 에 압축해 두었다가 입력에 어울리는 출력을 새로 생성합니다.

## 토큰이란 무엇인가?

사람은 문장을 단어와 의미 중심으로 읽지만, LLM은 문자열을 그대로 이해하지 못합니다. 먼저 입력된 문자열을 모델이 처리할 수 있는 작은 단위로 나누는데, 이 단위가 **토큰(token)** 입니다. 문자열을 토큰으로 나누는 프로그램을 **토크나이저(tokenizer)** 라고 합니다.

토큰은 단어와 같을 수도 있지만 항상 그런 것은 아닙니다.

- 자주 등장하는 짧은 단어는 하나의 토큰이 될 수 있습니다.
- 긴 단어나 드문 단어는 여러 토큰으로 나뉠 수 있습니다.
- 조사, 공백, 숫자, 문장 부호가 별도의 토큰이 될 수 있습니다.
- 코드에서는 변수 이름이나 연산자가 여러 토큰으로 나뉠 수 있습니다.

예를 들어 `인공지능을 공부한다.`라는 문장은 개념적으로 다음과 같이 나뉠 수 있습니다.

```text
[인공] [지능] [을] [ 공부] [한다] [.]
```

이것은 이해를 돕기 위한 예시입니다. 실제 분리 결과는 모델이 사용하는 토크나이저와 **어휘 집합(vocabulary)** 에 따라 달라집니다. 같은 문장도 모델마다 토큰의 개수가 다를 수 있습니다.

### 왜 글자나 단어를 그대로 사용하지 않을까?

모든 글자를 하나씩 처리하면 어떤 문자열도 표현할 수 있지만 문장이 너무 길어지고, 모델이 단어의 의미를 익히기 어려워집니다. 반대로 모든 단어를 하나의 단위로 저장하면 활용 가능한 단어의 수가 지나치게 많아지고, 처음 보는 단어를 처리하기 어렵습니다.

그래서 많은 LLM은 자주 쓰이는 문자열은 하나의 토큰으로 묶고, 드문 문자열은 더 작은 조각으로 나누는 방식을 사용합니다. 이를 통해 제한된 어휘 집합으로 신조어, 이름, 여러 언어와 코드까지 표현할 수 있습니다.

### 토큰은 모델 안에서 어떻게 처리되는가?

토크나이저는 각 토큰을 어휘 집합에 정의된 정수 ID로 변환합니다. 모델은 이 숫자를 그대로 의미로 해석하는 것이 아니라, 각 ID를 의미와 사용 관계를 나타내는 벡터인 **[임베딩(embedding)](../glossary/embedding.mdx)** 으로 바꿔 처리합니다.

```text
문자열 → 토큰 → 토큰 ID → 임베딩 → LLM → 다음 토큰의 확률
```

모델이 다음 토큰을 고르면 그 토큰을 기존 입력 뒤에 붙이고 다시 계산합니다. 생성형 LLM은 이 과정을 반복해 짧은 답부터 긴 글까지 만들어냅니다. 따라서 모델이 문장 전체를 한 번에 꺼내 놓는 것이 아니라, 앞에서 생성한 결과를 문맥으로 삼아 다음 토큰을 계속 선택한다고 볼 수 있습니다.

### 토큰 수가 중요한 이유

LLM이 한 번에 처리할 수 있는 입력과 출력의 범위는 글자 수가 아니라 주로 토큰 수로 정해집니다. 이를 **컨텍스트 윈도우(context window)** 라고 합니다.

토큰 수가 많아지면 다음과 같은 영향을 줍니다.

- 모델이 처리해야 할 문맥이 늘어나 응답 시간이 길어질 수 있습니다.
- API 서비스에서는 입력 토큰과 출력 토큰을 기준으로 비용을 계산하는 경우가 많습니다.
- 컨텍스트 윈도우의 한도를 넘은 내용은 한 번의 요청에 모두 넣을 수 없습니다.
- 같은 글자 수라도 언어, 숫자, 코드와 토크나이저에 따라 토큰 수가 달라질 수 있습니다.

토큰 단위로 글을 처리한다는 특성은 LLM이 글자 수를 정확히 세거나 단어 안의 철자 위치를 다루는 문제에서 실수할 수 있는 이유 중 하나이기도 합니다. 사람에게는 한 단어로 보이는 문자열이 모델에게는 여러 조각일 수 있기 때문입니다.

## LLM은 어떻게 학습하는가?

LLM의 학습 과정은 크게 다음처럼 이해할 수 있습니다.

1. 텍스트를 토큰 단위로 나눕니다.
2. 앞의 토큰을 보고 다음 토큰을 예측합니다.
3. 실제 토큰과 예측 결과의 차이를 계산합니다.
4. 차이가 줄어들도록 모델의 파라미터를 조정합니다.
5. 방대한 텍스트에 이 과정을 반복합니다.

이 과정을 **사전 학습(pre-training)** 이라고 합니다. 사전 학습을 마친 모델은 문법, 문체, 개념 사이의 관계와 여러 문제 해결 패턴을 폭넓게 익힙니다.

이후에는 사람이 작성한 질문과 모범 답변으로 추가 학습하거나, 사람의 선호도 피드백을 반영해 사용자의 지시를 더 잘 따르도록 조정할 수 있습니다. 이를 **지시 튜닝(instruction tuning)**, **정렬(alignment)** 등의 과정이라고 합니다.

실제로 사용자가 프롬프트를 입력하고 모델이 답을 생성하는 단계는 **추론(inference)** 이라고 부릅니다. 여기서 추론은 논리적 사고만을 뜻하지 않고, 학습된 모델로 결과를 계산하는 실행 과정 전체를 의미합니다.

## LLM의 발전 단계

현재의 LLM은 어느 날 갑자기 등장한 것이 아니라, 언어를 수치로 표현하고 문맥을 처리하는 방법이 발전하면서 만들어졌습니다.

### 1. 통계적 언어 모델

초기의 언어 모델은 앞에 나온 몇 개의 단어를 보고 다음 단어의 확률을 계산했습니다. 대표적으로 **[N-gram](../glossary/n-gram.mdx)** 방식이 있습니다.

구조가 단순하고 해석하기 쉽지만, 멀리 떨어진 단어의 관계를 반영하기 어렵고 한 번도 보지 못한 문장 조합에 약했습니다.

### 2. 신경망 기반 언어 모델

단어를 숫자 벡터로 표현하는 **[임베딩(embedding)](../glossary/embedding.mdx)** 과 [RNN](../glossary/rnn.mdx), LSTM 같은 신경망이 사용되기 시작했습니다.

통계적 방식보다 문맥과 단어 사이의 의미 관계를 잘 학습했지만, 문장이 길어질수록 앞의 정보를 유지하기 어렵고 학습을 병렬로 처리하기 힘들다는 한계가 있었습니다.

### 3. Transformer의 등장

2017년에 발표된 Transformer는 **[어텐션(attention)](../glossary/attention.mdx)** 을 이용해 문장 안의 토큰들이 서로 어떤 관련이 있는지 계산합니다.

RNN처럼 토큰을 반드시 순서대로 처리하지 않아도 되므로 대규모 병렬 학습에 유리했고, 긴 문맥의 관계도 더 효과적으로 다룰 수 있었습니다. 오늘날 대부분의 LLM은 Transformer 또는 그 변형을 기반으로 합니다.

### 4. 대규모 사전 학습과 확장

Transformer에 더 많은 데이터와 파라미터, 연산량을 투입하면서 하나의 모델이 요약, 번역, 질의응답, 코드 생성 등 여러 작업을 수행하기 시작했습니다.

특히 GPT-3는 작업마다 모델을 다시 학습하지 않아도 프롬프트에 지시나 몇 개의 예시를 제공하는 **[zero-shot, few-shot](../glossary/zero-shot-few-shot.mdx)** 방식으로 다양한 문제를 풀 수 있음을 보여주었습니다.

### 5. 지시를 따르는 대화형 모델

사전 학습만 한 모델은 문장을 이어 쓰는 데는 능숙하지만, 사용자의 의도를 따르거나 안전한 답을 하는 데는 부족할 수 있습니다.

지시 데이터와 사람의 피드백을 이용한 추가 학습이 적용되면서 질문에 답하고 대화를 이어가는 모델로 발전했습니다. 이 단계에서 LLM은 일반 사용자가 자연어로 활용하기 쉬운 도구가 되었습니다.

### 6. 도구를 사용하는 복합 시스템

최근의 AI 서비스는 LLM 하나만으로 모든 일을 처리하지 않습니다. 필요한 문서를 검색하는 **[RAG(Retrieval-Augmented Generation)](../glossary/rag.mdx)**, 계산기, 코드 실행, 웹 검색, 데이터베이스 같은 도구를 LLM과 연결합니다. 텍스트뿐 아니라 이미지와 음성도 함께 처리하는 **[멀티모달(multimodal)](../glossary/multimodal.mdx)** 모델도 널리 사용됩니다.

이때 검색이나 도구 사용은 LLM의 다음 토큰 예측 능력을 없애는 것이 아니라, 모델이 판단에 사용할 문맥과 외부 기능을 보완합니다. 즉, LLM은 전체 시스템의 언어 인터페이스이자 판단을 조율하는 구성 요소로 동작합니다.

## LLM이 잘하는 것

LLM은 언어 패턴을 바탕으로 내용을 **생성하고 변환하는 일**에 강합니다.

### 글을 만들고 다듬기

- 게시글, 이메일, 보고서의 초안 작성
- 맞춤법 교정과 문체 변경
- 긴 글의 요약과 핵심 내용 정리
- 번역과 쉬운 표현으로 바꾸기

### 형식을 바꾸기

- 자연어 요구사항을 표나 JSON 같은 구조로 변환
- 여러 문서에서 공통 항목 추출
- 정해진 양식에 맞춰 내용을 재구성

### 아이디어와 예시 만들기

- 주제에 대한 아이디어 확장
- 개념을 설명하는 비유와 예시 생성
- 여러 대안의 장단점 초안 작성

### 코드 작업 보조하기

- 짧은 코드와 테스트 코드 초안 생성
- 익숙하지 않은 코드 설명
- 오류가 발생할 만한 지점과 수정 방향 제안

LLM은 특히 사용자가 충분한 문맥과 원하는 출력 형식을 제공했을 때 더 안정적으로 동작합니다. 참고 자료, 예시, 제약 조건을 함께 주면 막연한 한 문장만 주었을 때보다 원하는 결과에 가까워집니다.

## LLM이 못하거나 주의해야 하는 것

LLM의 답은 자연스럽게 보인다는 것과 사실이라는 것이 별개입니다.

### 사실을 항상 보장하지 못한다

LLM은 그럴듯한 다음 토큰을 선택하기 때문에, 모르는 내용도 자신 있는 문장으로 만들어낼 수 있습니다. 존재하지 않는 인물, 논문, 함수나 출처를 제시하는 현상을 흔히 **환각(hallucination)** 이라고 부릅니다.

따라서 중요한 사실, 숫자, 인용문, 법률이나 의료 정보는 원문과 신뢰할 수 있는 출처로 다시 확인해야 합니다.

### 최신 정보와 비공개 정보를 저절로 알지 못한다

기본 모델이 아는 정보는 학습 데이터와 학습 시점에 제한됩니다. 오늘의 뉴스, 사내 문서, 개인 데이터는 검색 도구나 별도의 데이터 연결 없이 알 수 없습니다.

### 정확한 계산과 엄격한 규칙 수행에 약할 수 있다

산술 계산, 글자 수 세기, 복잡한 조건을 빠짐없이 적용하는 작업은 작은 실수에도 결과가 틀립니다. 계산기, 코드 실행기, 데이터베이스 같은 도구로 검증하는 편이 안전합니다.

### 긴 작업에서 일관성을 잃을 수 있다

입력이 모델이 한 번에 처리할 수 있는 **컨텍스트 윈도우(context window)** 를 넘으면 앞부분을 참고하지 못합니다. 범위 안에 있더라도 긴 문서의 모든 조건을 같은 비중으로 기억하거나 여러 단계의 계획을 끝까지 정확히 수행한다는 보장은 없습니다.

### 질문이 모호하면 의도와 다른 답을 만들 수 있다

LLM은 사용자의 숨은 목적을 완전히 알 수 없습니다. 대상 독자, 목적, 분량, 포함하거나 제외할 내용을 알려주지 않으면 일반적이지만 쓸모가 적은 답이 나올 수 있습니다.

## LLM을 잘 활용하는 방법

LLM은 정답을 보관한 백과사전보다 **빠르게 초안을 만들고, 주어진 정보를 정리하며, 도구 사용을 연결하는 협업자**에 가깝습니다.

- 목적과 독자를 구체적으로 알려줍니다.
- 필요한 배경 자료와 좋은 결과의 예시를 제공합니다.
- 큰 작업은 중간 결과를 확인할 수 있는 작은 단계로 나눕니다.
- 중요한 사실과 계산은 외부 출처나 도구로 검증합니다.
- 개인정보와 회사의 기밀 정보는 사용 중인 서비스의 데이터 정책을 확인한 뒤 입력합니다.

결국 LLM을 잘 쓰는 핵심은 모든 답을 그대로 믿는 것이 아니라, **생성과 판단은 LLM의 도움을 받고 검증은 적절한 도구와 사람이 맡는 것**입니다.

## 정리

LLM은 방대한 텍스트에서 언어 패턴을 학습하고, 주어진 문맥에 이어질 토큰을 예측해 결과를 생성하는 모델입니다.

글쓰기, 요약, 변환, 아이디어 생성처럼 언어를 다루는 작업에는 강하지만, 사실성, 최신성, 정확한 계산과 긴 작업의 일관성을 스스로 보장하지는 못합니다. Transformer와 대규모 사전 학습을 거쳐 범용성이 크게 높아졌고, 현재는 검색과 여러 도구를 결합한 시스템으로 발전하고 있습니다.
