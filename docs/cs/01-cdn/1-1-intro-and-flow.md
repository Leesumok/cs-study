# 1-1. CDN이란? CDN 작동

CDN은 **Content Delivery Network**의 약자입니다.

사용자와 원본 서버(origin server) 사이에 전 세계 여러 지역의 엣지 서버를 두고, 콘텐츠나 요청을 사용자에게 더 가까운 곳에서 처리하게 하는 네트워크입니다. 여기서 엣지 서버는 보통 **캐시 서버**, **PoP(Point of Presence)** 라고도 부릅니다.

은행 본점까지 매번 가지 않고 가까운 ATM에서 돈을 찾는 것과 비슷합니다. 원본 서버를 본점이라고 보면, CDN의 엣지 서버는 사용자의 가까운 위치에 놓인 ATM처럼 자주 필요한 응답을 더 가까이에서 제공합니다.

## CDN을 쓰는 이유

- 사용자와 서버 사이의 물리적 거리를 줄여 지연 시간을 낮춥니다.
- 정적 파일을 가까운 엣지 서버에서 내려주어 페이지 로딩을 빠르게 만듭니다.
- 원본 서버가 직접 처리해야 하는 요청과 대역폭을 줄입니다.
- 트래픽 급증이나 장애 상황에서 더 높은 가용성을 제공합니다.
- DDoS 방어, WAF, 봇 차단 같은 보안 기능을 함께 제공할 수 있습니다.

## 구성 요소

### Origin server

원본 서버는 콘텐츠와 API의 기준 데이터가 있는 서버입니다.

콘텐츠가 처음 만들어지는 곳이며, 엣지 서버에 캐시가 없거나 캐시가 만료되면 엣지 서버는 원본 서버로 요청을 전달합니다.

### Edge server

엣지 서버는 사용자 가까이에 배치된 CDN 서버입니다.

정적 파일을 캐시해두고, 사용자의 요청이 들어오면 가능한 한 원본 서버까지 가지 않고 바로 응답합니다. 캐시가 없거나 오래된 경우에는 원본 서버에서 새 응답을 받아오고, 정책에 따라 다시 캐시합니다.

### DNS server

DNS 서버는 사용자가 입력한 도메인을 어떤 CDN 엣지로 보낼지 결정하는 출발점입니다.

사용자가 서비스 도메인에 접근하면 DNS는 사용자의 위치, 네트워크 상태, 엣지 서버의 상태 등을 고려해 적절한 CDN 엣지 서버로 연결되도록 응답합니다.

## 기본 동작 흐름

1. 사용자가 `example.com` 같은 서비스 도메인에 접근합니다.
2. DNS가 가까운 CDN 엣지 서버로 사용자를 안내합니다.
3. 사용자의 요청이 엣지 서버에 도착합니다.
4. 엣지 서버에 캐시된 응답이 있으면 바로 반환합니다.
5. 캐시가 없거나 만료되었다면 엣지 서버가 원본 서버로 요청합니다.
6. 원본 서버의 응답을 사용자에게 전달하고, 캐시 가능한 응답은 엣지 서버에 저장합니다.
